SafetyInspect KZ заменяет бумажные чек-листы и Excel. Инспектор проводит проверку офлайн, система генерирует PDF-акт со ссылками на НПА РК и автоматически назначает корректирующие мероприятия.

Проблема

Потеря данных

Возможности

📝

Цифровые чек-листы

5 разделов по НПА РК через единый JSON-движок. Ответ «Нет» — автоматически требует комментарий и фото. Ссылка на НПА подставляется из данных.

  • Охрана труда и здоровья
  • Пожарная безопасность
  • Электробезопасность
  • Транспортная безопасность
  • Промышленная безопасность
📄

PDF-акт за 5 секунд

Автоматическая генерация акта с нарушениями, фото и ссылками на НПА. Отправка по Email / WhatsApp / Telegram.

📡

100% офлайн

Работает без интернета на территории завода. При появлении сети — автосинхронизация с сервером.

✅

Корректирующие мероприятия

Автоназначение задач по нарушениям. Push-уведомления за 3 дня до срока. Жизненный цикл: Открыто → В работе → Закрыто.

Как это работает

От начала инспекции до закрытия нарушения — автоматически

1

Инспектор входит в приложение

Авторизация по логину/паролю (JWT). Выбирает объект и раздел чек-листа.

→
2

Проводит инспекцию

Отвечает на вопросы, фотографирует нарушения. Работает офлайн — данные в SQLite.

→
3

Генерирует акт

PDF формируется за <5 секунд с нарушениями, фото и ссылками на НПА РК.

→
4

Исполнитель устраняет

Получает задачу с дедлайном, загружает фотодоказательство устранения.

→
5

Руководитель контролирует

Видит все KPI на дашборде. При появлении сети данные синхронизируются автоматически.

Жизненный цикл нарушения

Открыто Нарушение зафиксировано
→
В работе Исполнитель принял
→
На проверке Заявлено выполнение
→
Закрыто Инспектор подтвердил

При истечении срока статус автоматически меняется на Просрочено с ежедневными уведомлениями

План разработки

6–7 недель до первой реальной инспекции

Нед 1

Подготовка

Стек, проект, навигация, CI/CD

Нед 2

Авторизация

Вход, роли, главный экран с 5 разделами

Нед 3–4

Чек-листы

JSON-движок, все 5 разделов, фото, офлайн-режим

Нед 5

PDF и сервер

Генерация акта, API, офлайн-очередь синхронизации

Нед 6–7

Дашборд + QA

Виджеты, тестирование на заводе, правки

Роли пользователей

👷

Инспектор по ОТ/ПБ

Основной пользователь, работает в поле. Проводит инспекцию, фиксирует нарушения, генерирует акт.

👔

Руководитель по БиОТ

Смотрит дашборд, контролирует мероприятия. Просматривает отчёты, утверждает мероприятия.

🔧

Ответственный исполнитель

Устраняет нарушения. Получает задачи с дедлайнами, загружает фотодоказательства выполнения.
